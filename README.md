# 易占サイト（サーバー側ロジック版）— Vercelお試しデプロイ手順

このプロジェクトは、占いの判定ロジック・64卦データ・解釈テンプレートを
すべて `api/divine.js`（サーバー側）に隠し、ブラウザ側（`public/index.html`）には
見た目の制御だけを残した構成です。

ページのソースを表示しても、コインアニメーションの仕組みは見えますが、
64卦の対応表・解釈テンプレートの中身は一切見えません。

---

## フォルダ構成

```
iching-vercel-project/
├── api/
│   └── divine.js      ← サーバー側のロジック（ブラウザには送られない）
├── public/
│   └── index.html     ← ブラウザ側（結果を表示するだけ）
├── vercel.json         ← Vercelの設定
└── package.json
```

## デプロイ手順（無料お試し）

### 1. Vercelアカウントを作る
https://vercel.com/signup にアクセスし、GitHubアカウントでログインするのが簡単です。

### 2. このフォルダをGitHubリポジトリにする
お使いのPC上のターミナルで、このフォルダに移動して次を実行します。

```bash
git init
git add .
git commit -m "易占サイト 初回コミット"
```

GitHub上で新しいリポジトリを作成し、表示される指示に従って push します。

```bash
git remote add origin https://github.com/【あなたのユーザー名】/【リポジトリ名】.git
git branch -M main
git push -u origin main
```

### 3. Vercelでインポートする
Vercelのダッシュボードで「Add New...」→「Project」を選び、
先ほどpushしたGitHubリポジトリを選択して「Deploy」を押します。
設定はすべて初期値のままで問題ありません（Build Command等は空のままでOK）。

### 4. 発行されたURLで確認する
デプロイが完了すると `https://【プロジェクト名】.vercel.app` のようなURLが発行されます。
このURLにアクセスし、実際にコインを投げて占えるか確認してください。

### 5. ページのソースを確認してみる
ブラウザで「ページのソースを表示」（または開発者ツール）を開き、
64卦のデータや解釈テンプレートが見えないことを確認してみてください。
見えるのは `fetch('/api/divine', ...)` という、サーバーに問い合わせている部分だけです。

## 注意点

- **Vercelの無料プラン（Hobby）は個人の非商用利用が前提**です。
  お試し・動作確認の段階では問題ありませんが、実際にこの占いサイトで
  鑑定の販売など収益を得る運用に進める場合は、Proプラン（月額20ドル程度）への
  切り替えが必要になります。
- 今回のコードは「テーマ選択」「自由記述の問い」の2モードに対応した
  ¥500鑑定相当の体験です。無料診断（問い固定・卦名と一言のみ）は、
  `mode: "free"` を指定するとサーバー側で自動的に簡易な結果（変爻の詳細や
  テーマ別解釈を含まない）が返るようになっています。
- ¥500・¥20,000それぞれの申し込みボタン（`<a href="#">`の部分）は、
  まだ実際の申し込みページに接続されていないダミーのままです。
